Evaluation Results

Detailed findings from our heuristic evaluation

Heuristic Evaluation Workbook

This section presents a summary of the usability evaluation conducted on the Library Management System. The primary objective of this evaluation was to assess the system’s interface with regard to usability, focusing on factors such as effectiveness, efficiency, and user satisfaction. Several usability issues were identified, affecting different areas of the system's design and user interaction flow.

The evaluation process followed Jacob's 10 Heuristic Evaluation method, a widely recognized framework that employs ten usability principles to systematically identify and assess interface design issues. These principles enabled a structured analysis of the system, revealing both minor inconsistencies and more significant usability barriers that could impede user performance and overall experience.

The evaluation has revealed critical insights into areas of the Library Management System that require improvement. The findings provide a valuable foundation for guiding future design revisions, with the goal of enhancing usability and ensuring a more seamless and user-centered experience.

Usability Issues

The following issues were identified during our heuristic evaluation:

Issue #1 Visibility of System Status

Lack of Immediate Feedback and Navigation Cues

The system fails to provide timely and clear feedback for user actions such as logging in or deleting records. It does not highlight the current location within the interface, and lacks confirmation messages, which can leave users uncertain about what has occurred or what steps to take next.

Issue #2 Match Between System and Real World

Navigation and Date Selection Usability Issues

The date picker combines years and months in a single list, which disrupts the expected order and makes it harder for users to select dates. Presenting years separately and in sequence would make the selection process more intuitive. Additionally, the burger icon is not functioning, preventing users from accessing the main navigation menu. This limits usability and should be fixed to ensure smooth and efficient navigation.

Issue #3 User Control and Freedom

No Undo, Cancel, or Error Recovery Options

Users have no way to reverse or cancel actions. Mistakes like accidental deletion or form errors force them to restart entire processes, creating frustration and a sense of being trapped within the system’s rigid workflows.

Issue #4 Consistency and Standards

Confusing Interface and Data Inconsistencies

There are frequent mismatches between displayed data and actual records (e.g., showing 8 borrowed books but only listing 4), along with inconsistent use of symbols and terminology. These inconsistencies undermine trust and usability.

Issue #5 Error Prevention

Inadequate Safeguards Against User Errors

Many actions proceed without confirmation, and the system provides vague or missing guidance when users make input mistakes. Error prevention is weak, which increases the likelihood of user errors going unnoticed or unresolved.

Issue #6 Recognition Rather Than Recall

Icons and Navigation Rely Too Much on Memory

The system’s icons are visually indistinct and often misleading, requiring users to remember their functions. Navigation is not intuitive, and context is often missing from forms or pages, increasing user effort and confusion.

Issue #7 Flexibility and Efficiency of Use

No Support for Efficient or Advanced Use

There are no batch actions, shortcuts, or user preferences. Users must perform repetitive tasks manually, with no way to speed up their workflow, especially frustrating for administrators managing large sets of data.

Issue #8 Aesthetic and Minimalist Design

Gear Icon Spins Unnecessarily

The gear icon kept spinning on the settings page when it shouldn’t have. This continuous animation created unnecessary visual clutter. It distracted users and made the interface feel less polished.

Issue #9 Help Users Recognize, Diagnose, and Recover from Errors

Technical Error Messages Without Guidance

When errors occur, the system displays confusing technical terms (like “Undefined array key”) that don’t explain what went wrong or how to fix it. Users are given no clear path to recovery, causing frustration and delays.

Issue #10 Help and Documentation

No Help Resources or In-System Guidance

The system provides no help section, tooltips, instructions, or user documentation. Users are left to figure things out entirely on their own, which can be especially problematic when encountering errors or unfamiliar features.

Conclusion

The system has many problems that make it hard to use, especially for new users. It doesn’t give clear feedback, uses confusing words and icons, and lacks important features like undo, help guides, and shortcuts. Errors are often unclear and the design feels messy and overwhelming.

These issues can cause users to make mistakes, feel lost, and waste time doing simple tasks. To improve, the system needs to be more user-friendly, clear, and supportive at every step.
