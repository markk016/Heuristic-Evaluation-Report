Proposed Improvements

Suggested enhancements based on our heuristic evaluation

Overview

Based on our heuristic evaluation using Jakob Nielsen's 10 Usability Heuristics, we have developed a comprehensive set of proposed improvements to address the identified usability issues. These suggestions are grounded in HCI principles and aim to enhance the overall user experience of the system.

Improvement for Issue #1

Visibility of System Status

The system fails to provide timely and clear feedback for user actions. Users are often left uncertain about whether their actions have been completed successfully or what their current location is within the system.

Improvement 1.1

Current Design

When deleting member accounts it has no confirmation feedback.

Proposed Design

The system should show a message like "Member account deleted successfully!" so users know the action worked without any confusion.

Improvement 1.2

Current Design

The system does not clearly indicates where the user is currently on and what the user is dealing with, there’s no pointers or feedback or anything that could indicate what the user should do on a particular page, but basically all of it, and this is crucial especially for a library system.

Proposed Design

Add confirmation messages for every changes wether it be for changing names, adding new admins or books.

Improvement for Issue #2

Match Between System and Real World

The system uses terminology and concepts that don't match real-world library conventions, creating confusion for users familiar with standard library operations.

Improvement 2.1

Current Design

The menu button or burger menu button doesn't not work at all.

Proposed Design

Make the burger menu work because it loses the function all together.

Improvement 2.2

Current Design

The date picker mixes years and months in one list, creating confusion and breaking the expected chronological order. Years should be presented separately and sequentially to match familiar date selection patterns.

Proposed Design

Separate year and month selection by listing years clearly and chronologically first, then displaying months for the chosen year. This improves clarity and aligns with standard date picker behavior.

Improvement for Issue #3

User Control and Freedom

The system lacks essential mechanisms for users to undo actions, cancel operations in progress, or easily navigate back from unwanted states, forcing users to complete unwanted actions or start over entirely.

Improvement 3.1

Current Design

There is no undo or redo, no even the exit links, and there is a cancel action or confirmation first if the user really wants to cancel something, but what if the user accidentally just clicked it mid-fill-out of something and it all just disappear and the user would have to re-write it again from the beginning, no back ups or automatic save at all which could be very frustrating.

Proposed Design

Add auto save so the user won't have to fill out again if they make a mistake.

Improvement 3.2

Current Design

There are no visible "undo" or "back" options within the interface, which could make users feel trapped if they make a mistake.

Proposed Design

Implement clear cancel buttons, confirmation dialogs for important actions, and breadcrumb navigation to allow users to easily backtrack or undo actions.

Improvement 3.3

Current Design

After submitting the form with a duplicated ID, the system shows an error message but then redirects back to the member list instead of staying on the form. These forces restarting the process to fix the error.

Proposed Design

The system should display the error message and keep the form open, allowing correction without losing entered information or needing to start over.

Improvement for Issue #4

Consistency and Standards

The system lacks consistency in its interface elements, layout, and interaction patterns, and fails to follow established conventions, forcing users to learn different approaches for similar actions across the application.

Improvement 4.1

Current Design

The login button shows a checkmark symbol instead of a word, which doesn't follow common UI standards.

Proposed Design

Use standard buttons like “Login” instead of just icons.

Improvement 4.2

Current Design

The dashboard displays "Book Borrowed" and the misspelled "Book Retured," which are inconsistent with the sidebar labels "Borrowed Books" and "Returned Books." This inconsistency can cause confusion and reduce interface credibility.

Proposed Design

Use consistent terminology across the interface by updating all labels to "Borrowed Books" and "Returned Books," and correct the spelling mistake.

Improvement 4.3

Current Design

The homepage indicated that there were 8 borrowed books, but upon checking the list, only 4 were actually displayed.

Proposed Design

Ensure that the number of borrowed books displayed on the homepage accurately reflects the actual list of books shown.

Improvement for Issue #5

Error Prevention

The system lacks proactive measures to prevent errors, allowing users to make mistakes that could be avoided through better design, and failing to confirm potentially destructive actions before they are executed.

Improvement 5.1

Current Design

Entering an incorrect username or password displays the vague message "Access Denied", without indicating the specific problem.

Proposed Design

Show specific messages like "Incorrect password or password" or "Username not found" along with guidance to resolve the issue.

Improvement 5.2

Current Design

The system inconsistently requests confirmation from users before performing actions. While some actions prompt for confirmation, many others do not and proceed automatically.

Proposed Design

Implement consistent confirmation prompts for all critical actions to prevent unintended changes.

Improvement for Issue #6

Recognition Rather Than Recall

The system forces users to remember information across different parts of the interface instead of making options visible and providing contextual information, increasing cognitive load and error rates.

Improvement 6.1

Current Design

The icons for books, borrowed books, and returned books all look the same, making it difficult to distinguish between them at a glance.

Proposed Design

Use distinct and meaningful icons—such as a plain book for total books, a book with an arrow out for borrowed books, and a book with an arrow in for returned books.

Improvement for Issue #7

Flexibility and Efficiency of Use

The system offers a one-size-fits-all interface that doesn't accommodate different user expertise levels or provide shortcuts for frequent tasks, forcing experienced users to follow the same lengthy processes as beginners.

Improvement 7.1

Current Design

There are no shortcuts or efficient paths to make the system work smoothly and hassle-free for both inexperienced and experienced users.

Proposed Design

Allowing drag-and-drop file uploads will streamline the process by reducing the number of steps needed to add files. This enhancement improves efficiency for experienced users while maintaining ease of use for beginners.

Improvement 7.2

Current Design

There are no visible shortcuts or accelerators for experienced users to speed up their workflow.

Proposed Design

Implement keyboard shortcuts, batch operations (e.g., check out multiple books at once), and customizable dashboard widgets to allow power users to work more efficiently.

Improvement for Issue #8

Aesthetic and Minimalist Design

The interface is cluttered with unnecessary information, decorative elements, and redundant features that distract users from their primary tasks and make important information harder to find.

Improvement 8.1

Current Design

The gear icon kept spinning on the settings page, even though it shouldn't have. This created unnecessary visual clutter.

Proposed Design

The gear icon should either stay static or be removed to keep the design clean and avoid distracting the user.

Improvement for Issue #9

Help Users Recognize, Diagnose, and Recover from Errors

The system displays cryptic, technical error messages that don't help users understand what went wrong or how to fix the problem, leaving them confused and unable to recover from errors effectively.

Improvement 9.1

Current Design

When deleting an admin account, there's an error saying "Undefined array key 'adminid”. It doesn’t provide any useful feedback to the user about what went wrong or how to resolve it.

Proposed Design

Instead of a PHP error, the system should show "Unable to delete admin account. Please try again." to give users clear feedback and help them take action.

Improvement for Issue #10

Help and Documentation

The system lacks adequate help resources and documentation, providing no built-in guidance for users who need assistance understanding features or completing tasks.

Improvement 10.1

Current Design

There are no visible help resources, documentation links, or contextual guidance within the interface.

Proposed Design

Add a help section from every page, include contextual help icons next to complex features, and provide tooltips for form fields to guide users through processes like adding new books or managing loans.

Improvement 10.2

Current Design

The system doesn’t have a Help Section or Documentation, so when I encountered an error, I had no way to find guidance or troubleshoot the issue.

Proposed Design

Create an integrated, searchable knowledge base accessible directly from the system. Organize content by common tasks and user roles rather than system features. Include step-by-step guides, FAQs, troubleshooting tips, and best practices. Implement powerful search functionality with filters and suggested results. Ensure content is written in plain language with screenshots and examples. Allow users to access the knowledge base without leaving their current screen.
